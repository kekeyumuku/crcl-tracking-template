# CRCL 跟踪模板

> 用途：不是预测股价，而是持续判断 **CRCL 当前仍是利差驱动发行商，还是正在升级为支付基础设施平台**。

> 核心原则：**不改模型，不落卡。**

## 适合谁用

- 跟踪 Circle / CRCL 的投资者
- 研究稳定币与支付基础设施的人
- 想区分“利润波动”和“业务身份迁移”的研究者

## 这份模板主要解决什么问题

很多跟踪笔记容易把这些东西混在一起：

- 季度利润变化
- 利率周期变化
- 市场情绪
- 监管叙事
- 平台化进展

这份模板的目标，是把它们拆开，只追踪真正影响以下三件事的信息：

1. **底盘有没有变化**
2. **上限有没有被验证**
3. **公司身份有没有迁移**

## 使用方法

每次看到和 CRCL / USDC / 稳定币监管 / 企业支付 adoption 相关的新信息，就先按下面的最小版速填模板记一条。

只有当这条信息真的改变了模型，再升级为正式跟踪卡。

参考示例：`examples/example-fill.zh.md`

## 八个核心指标

前四项主要判断 **当前底盘**，后四项主要判断 **长期上限**。

### A. 底盘
1. **USDC 总流通规模（Circulation）**
2. **USDC 增长来源（Growth Quality）**
3. **储备收益率 / 利率敏感度（Rate Sensitivity）**
4. **渠道分成比例 / 收入留存能力（Revenue Retention）**

### B. 上限
5. **非利差收入占比（Non-Spread Revenue）**
6. **USDC 市场份额变化（Market Share）**
7. **监管进展与合规地位（Regulatory Position）**
8. **企业 / 支付场景真实落地（Enterprise / Payments Adoption）**

## 口径提示

### 增长质量
- **低**：主要来自短期交易活跃、一次性流量或脉冲式需求
- **中**：有多场景扩张，但持续性还未验证
- **高**：更多来自企业、支付、跨境等更稳定需求，并具备延续迹象

### 平台化收入迹象
通常指以下收入或能力开始出现验证：
- API / 服务费
- 企业账户能力
- 支付编排或清结算能力
- 基于网络位置获得的持续 fee take

### 真实业务流程落地
不是“宣布合作”就算，而要看是否进入：
- 收款 / 付款 / 结算流程
- 对账 / treasury / 资金调度流程
- 持续交易量或可重复收入链路

## 先判断：这条信息值不值得正式落卡？

只有会改变以下三类判断的信息，才值得正式写进 CRCL 跟踪卡：

- **改底盘**：流通规模、增长来源、利率敏感度、渠道分成发生实质变化
- **改上限验证**：出现更强的平台化收入迹象、企业 / 支付真实落地、监管地位前进、关键场景份额提升
- **改竞争格局**：Circle 的领先优势被验证，或关键优势被削弱 / 复制

以下信息通常**不正式落卡**，只做口头判断即可：

- 纯叙事
- 纯路线图
- 纯情绪
- 没有数据或落地支撑的愿景表达

## 判读原则（速记）
- **看底盘**：流通规模、增长来源、利率、分成
- **看上限**：非利差收入、份额、监管、企业落地

## 常见误判提醒
- 把短期 USDC 增长误判成长期平台化验证
- 把监管友好表述误判成真实 adoption
- 把合作公告误判成已进入真实支付流程
- 把利率环境改善误判成 company-specific moat strengthening
- 把估值叙事变化误判成业务结构已变化
